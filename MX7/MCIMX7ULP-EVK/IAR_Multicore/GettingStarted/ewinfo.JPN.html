Getting Started

DESCRIPTION
===========
   This example project shows how to use the IAR Embedded Workbench for ARM
  to develop code for NXP MCIMX7ULP-EVK board. It shows a basic use of the
  GIC, the Low Power Periodic Interrupt Timers (LPIT) and the LPUART modules
  in single-core (standalone) and multi-core debug sessions.

COMPATIBILITY
=============
  The example project is compatible with NXP MCIMX7ULP-EVK board with MCIMX7ULP.
  By default, the project is configured to use the I-Jet JTAG interface.

CONFIGURATIONS
===============

Project: cortex_m4

  OCRAM_STANDALONE
      Intended for debug in the internal RAM in standalone debug session for
      the CM4 core.

  OCRAM_AMP_MULTICORE
      Intended for debug in the internal RAM in AMP multicore debug session.
      It loads automatically OCRAM_AMP_SLAVE binary file for
      the core B (CA7). OCRAM_AMP_SLAVE configuration should be built
      before building this configuration.

  OCRAM_DDR_AMP_MULTICORE
      Intended for debug the CM4 application in the internal RAM and debug
      the core B (CA7) application in the external DDR memory in AMP multicore
      debug session. It loads automatically DDR_AMP_SLAVE binary file for
      the core B (CA7). DDR_AMP_SLAVE configuration should be built
      before building this configuration.

Project: cortex_a7

  OCRAM_STANDALONE
      Intended for debug in the internal RAM the core B (CA7) application,
      in standalone debug session.

  OCRAM_AMP_SLAVE
      Not intended for standalone debug sessions.
      It is loaded automatically by the OCRAM_AMP_MULTICORE configuration.

  DDR_STANDALONE
      Intended for debug in the external DDR memory the core B (CA7) application,
      in standalone debug session.

  DDR_AMP_SLAVE
      Not intended for standalone debug sessions.
      It is loaded automatically by the OCRAM_AMP_MULTICORE configuration.

Note: When starting the slave project in multi-debug sessions, you will notice a
      warning "Failed to set breakpoint for "Stack window trigger", because the
      core is not yet enabled. You can disable this message by disabling the
      "Stack window trigger" breakpoint. Navigate to Tools->Options->Stack and
      and disable the "Stack pointer(s) not valid until program reaches:" option.

GETTING STARTED
===============
  1) Open the GettingStarted workspace and select a configuration.

  2) Connect the debugger probe and power the board.

  3) Connect the board's UART-TO-USB port (J11) to a PC using micro USB cable.
     Two new COM ports will appear on the PC. You should use the two COM ports named
     "Silicon Labs Dual CP210x USB to UART Bridge: Enhanced COM Port" - COM port with
     lower number for connection to "Cortex-M4" application and the COM port with higher
     number for connection to the "Cortex-A7" application. Check this by opening the
     Device Manager -> Ports(COM&LPT).

  4) On the PC open a terminal program for the each port with the following
     settings: 115200 bps, 8 data bits, no parity, 1 stop bit.

  5) Press CTRL+D or use Download and Debug button to start a debug session.

  6) Press F5 or use the GO button to run the example. A start message and the
     counted seconds will be printed on the terminal for each port.

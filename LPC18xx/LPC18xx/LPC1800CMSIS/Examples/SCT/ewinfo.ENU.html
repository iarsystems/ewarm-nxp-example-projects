SCT counter example

Example description:
	Purpose:
		This example describes how to use SCT to count an input pulse supply to
		one of its CTIN pins
	Process:
		First, to install and understand how to integrated Fizzim auto generating
		C code file into Keil, please read ../tools/doc.
		
		This example capture a rising edge for 4 times, each time store the current
		count value and toggle a pin. After finished, stop and display the result.
							
Directory contents:
	Fizzim tools to translate *.fzm to C code. 
	\EWARM: includes IAR project and configuration files 
	lpc18xx_libcfg.h: Library configuration file - include needed driver library for this example 
	Sct_SimpleCapture.c: Main program

How to run:
	Hardware configuration:		
		This example was tested on:
			Embedded Artists LPC1800 Evaluation board version 2.3
			Hitex LPC1800 Board

		Serial display configuration: (e.g: TeraTerm, Hyperterminal, Flash Magic...) 
			– 115200bps 
			– 8 data bit 
			– No parity 
			– 1 stop bit 
			– No flow control 
			- Use USB cable connect to board's UART1->USB port
			
		Pulse generator:
			- Connect P7_0 (GPIO3_8 pin 1 JP19) to PF_8 (CTIN_2 pin 20 X5)
						
	Running mode:
		RAM mode:   This example can be run on RAM mode with debugger. 
					All files must be build to .elf file, this file will be loaded into RAM through a 
					debugger tool before running 
		ROM mode:   This example can be run on EXFLASH mode with debugger.

	Step to run:
		- Step 1: Build example.
		- Step 2: Burn hex file into board (if run on ROM mode)
		- Step 3: Connect UART1->USB port on this board to USB port on your computer
		- Step 4: Configure hardware and serial display as above instruction 
		- Step 5: Run example and observe data from serial display or CTOUT[0] by oscilloscope		
		
Tip:
	- Open \arm\examples\NXP\LPC18xx\LPC1800CMSIS\Examples\SCT\SCT.eww project file to run example on IAR

SCT output control example
	
Example description:
	Purpose:
		This example describes how to use SCT to toggle its CTOUT pins
		when there is a match timer occur.
	Process:
		First, to install and understand how to integrated Fizzim auto generating
		C code file into Keil, please read ../tools/doc.
		
		This example serially set a SCT output pin high in a pre-programmed time.
		When counter reaches limit, this pin is set low, other pin is set high like:
		   All pin low -> CTOUT0 high -> CTOUT1 high -> CTOUT2 high -> CTOUT3 high
		-> All pin low -> CTOUT0 high -> ....
		
Directory contents:
	Fizzim tools to translate *.fzm to C code. 
	\EWARM: includes IAR project and configuration files 
	lpc18xx_libcfg.h: Library configuration file - include needed driver library for this example 
	Sct_SimpleMatch.c: Main program

How to run:
	Hardware configuration:		
		This example was tested on:
			Hitex LPC1800 Board
			
	Running mode:
		RAM mode:   This example can be run on RAM mode with debugger. 
					All files must be build to .elf file, this file will be loaded into RAM through a 
					debugger tool before running 
		ROM mode:   This example can be run on EXFLASH mode with debugger.


	Step to run:
		- Step 1: Build example.
		- Step 2: Burn hex file into board (if run on ROM mode)
		- Step 3: Run example and observe CTOUT[0...4] signals by oscilloscope
		
Tip:
	- Open \arm\examples\NXP\LPC18xx\LPC1800CMSIS\Examples\SCT\SCT.eww project file to run example on IAR

NVIC Priority example

Example description:
	Purpose:
		This example describes how to configure NVIC priority grouping for testing 
		tail-chaining/Late-arriving interrupt mode.
	Process:
		This example uses 2 interrupts as IRQ channels.
		Defining "SAME_GROUP" to chose interrupt mode test.
			- Define SAME_GROUP: Tail-chainning interrupt testing
			WIC interrupt is assigned group-priority = 1, sub-priority = 7
			DAC interrupt is assigned group-priority = 1, sub-priority = 2
			Two IRQ channels has same group.So, new ISR can not pre-empt previous interrupt 
			when it's executing even if new ISR has higher priority than curren ISR
			
			- Undefine SAME_GROUP: Late-arriving interrupt testing.
			WIC interrupt is assigned group-priority = 0, sub-priority = 1
			DAC interrupt is assigned group-priority = 1, sub-priority = 1
			WIC interrupt has higher group-priority than DAC interrupt, so WIC interrupt can 
			pre-empt DAC interrupt when it's executing
		
		In this example, WIC interrupt occurs when connecting WAKEUP0 pin to 3.3V. 
		WIC interrupt will blink USB0_IND1 LED (D10) 5 times.
		
		DAC interrupt occurs periodly and toggles USB1_IND1 LED (D9) after 1000 interrupt times.	
					
Directory contents:
	\IAR:	includes IAR project and configuration files
	lpc18xx_libcfg.h: Library configuration file - include needed driver library for this example 
	Nvic_Priorities.c: Main program

How to run:
	Hardware configuration:		
		This example was tested only on:
			Hitex LPC1850EVA-A4-2 Board
		Jumper settings:
			- JP3: close
			- JP4: close
			- SV6: pin 15-16 is open
	Running mode:
		>IAR project:
			Built the project (F7)
			Internal SRAM mode:	This example can run on Internal SRAM mode.
				Download and debug (CTRL + D)
				Seclect Device Cortex M4 (if any)
			Hitex Flash mode: This example can run on Hitex Flash (external parallel flash SST39X320X), SPIFI quad SPI flash (with or without debug)
				Select "Project->Download->Download active application" to burn into flash

	Step to run:
		- Step 1: Build example.
		- Step 2: Burn hex file into board (if run on Hitex Flash mode)
		- Step 3: Run example
				+ At the beginning, the DAC interrupt will normally happen and blink the USB1_IND1 LED (D9) in 1Hz
				+ If WIC interrupt is occurred (by connecting WAKEUP0 pin to 3.3V), USB0_IND1 LED (D10) will blink 
				5 times 
				
			- Test tail-chaining interrupt mode:
				When USB1_IND1 LED (D9) is blinking (under the effect of DAC interrupt), if connecting WAKEUP0 pin 
				to 3.3V the WIC interrupt will be generating to blink the USB0_IND1 LED (D10) 5 times right after 
				an DAC interrupt is finished 
				
			- Test late-arriving interrupt mode:
				When USB1_IND1 LED (D9) is blinking (under the effect of DAC interrupt), if connecting WAKEUP0 pin
				to 3.3V the WIC interrupt will be generating to blink the USB0_IND1 LED (D10) 5 times immediately
				without caring about the completion of the DAC interrupt. In this case, the WIC interrupt does 
				pre-empt the DAC interrupt
				After this WIC is completed, the DAC interrupt will re-start to blink the USB1_IND1 LED (D9) 
				continuously because its interrupt occurence.

Vector Table Relocation example

Example description:
	Purpose:
		This example describes how to relocation vetor table.
	Process:
		Vector Table will be remapped at new address VTOR_OFFSET.
		In Hitex Flash mode: Vector Table will be initialized at 0x1C000000
		In Internal SRAM mode: Vector Table will be initialized at 0x10000000
		In SPIFI 64MB Debug mode: Vector Table will be initialized at 0x14000000
		In SPIFI 128MB mode: Vector Table will be initialized at 0x80000000                
		So, we need copy vector table from init address to new address.
		Install timer interrupt printout a message every second.
		If VT remapping is successful, a message is secondly printed by timer interrupt.
					
Directory contents:
	\IAR:	includes IAR project and configuration files
	lpc18xx_libcfg.h: Library configuration file - include needed driver library for this example 
	Nvic_VectorTableRelocation.c: Main program

How to run:
	Hardware configuration:		
		This example was tested on:
			Hitex LPC1850EVA-A4-2 Board				
				
	Serial display configuration: (e.g: TeraTerm, Hyperterminal, Flash Magic...) 
			– 115200bps 
			– 8 data bit 
			– No parity 
			– 1 stop bit 
			– No flow control 
	
	Running mode:
		>IAR project:
			Built the project (F7)
			Internal SRAM mode:	This example can run on Internal SRAM mode.
				Download and debug (CTRL + D)
				Seclect Device Cortex M4 (if any)
			Hitex Flash mode: This example can run on Hitex Flash (external parallel flash SST39X320X), SPIFI quad SPI flash (with or without debug)
				Select "Project->Download->Download active application" to burn into flash

	Step to run:
		- Step 1: Build example.
		- Step 2: Burn hex file into board (if run on Hitex Flash mode) 
		- Step 3: Connect UART0 port on this board to COM port on your computer
		- Step 4: Configure hardware and serial display as above instruction 
		- Step 5: Run example and observe data from serial display

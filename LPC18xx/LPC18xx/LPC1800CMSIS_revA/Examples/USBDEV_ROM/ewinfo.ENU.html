USB ROM Composite(MassStorage, HID and DFU) example

Example description:
	Purpose:
		This example describes how to write a USB Composite(MassStorage, HID and DFU) 
		application using USB ROM driver on LPC18xx.


Directory contents:
	\IAR:	includes IAR project and configuration files
	
	usbd_dfu.c: contains USB DFU function
	usbd_msd_mem.c : contains USB MSC function and memory device definition
	usbd_hid.c : contains USB HID function
	composite_usbdesc.c : contains usb device descriptor.

How to run:
	Hardware configuration:		
		This example was tested on:
			Hitex LPC1800 Development Board
	
	Running mode:
		>IAR project:
			Built the project (F7)
			Internal SRAM mode:	This example can run on Internal SRAM mode.
				Download and debug (CTRL + D)
				Seclect Device Cortex M4 (if any)
			Hitex Flash mode: This example can run on Hitex Flash (external parallel flash SST39X320X), SPIFI quad SPI flash (with or without debug)
				Select "Project->Download->Download active application" to burn into flash

	Step to run:
		- Step 1: Build example.
		- Step 2: Burn hex file into board (if run on Hitex Flash mode)
		- Step 3: Configure hardware as above instruction 
		- Step 4: Run example
		- Step 5: Connect the USB0 port to PC with USB cable.
		- Step 6: Testing HID interface
  				- On PC launch C:\Keil\ARM\Utilities\HID_Client\Release\HIDClient.exe
    			- Select the "LPC18xx" device in the drop down box
    			- Now set check boxes next to "Outputs (LEDs)"
    			- The out report recived by the device loop-backed to PC. Hence the checkbox 
    			next to "Inputs (Buttons)" should look same a "Outputs".
			- Testing Mass storage interface
  				- On PC a new unformatted drive with 32KB size should appear.
    			- Format the drive and copy images to the drive.
			- Testing DFU interface
			  - On PC launch DFU download utility.
		
		(Pls see "LPC18xx Example Description" document - chapter "Examples > USBDEV_ROM > USBComposite"
		for more details)

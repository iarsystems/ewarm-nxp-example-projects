Example description

  
WDT interrupt mode example

Example description:
	Purpose:
		This example describes how to use WDT to generate interrupt after a specific time.
	Process:
		In this example, WDT disables reset function, WDT just generates interrupt when the 
		Watchdog times out. 
		WDT value is frequently fed by Systick_Handler().
		Open Terminal screen, press 1 to disable feeding, press 2 to enable feeding.
		If 1 is pressed, after WDT time out, "Warning...watchdog timeout!" is frequently displayed.
		
Directory contents:
	\IAR:	includes IAR project and configuration files
	Wdt_Interrupt.c: Main program

How to run:
	Hardware configuration:		
		This example was tested on:
			Hitex LPC1800 Board

		Serial display configuration: (e.g: TeraTerm, Hyperterminal, Flash Magic...) 
			– 115200bps 
			– 8 data bit 
			– No parity 
			– 1 stop bit 
			– No flow control 
	
	Running mode:
		>IAR project:
			Built the project (F7)
			Internal SRAM mode:	This example can run on Internal SRAM mode.
				Download and debug (CTRL + D)
				Seclect Device Cortex M4 (if any)
			Hitex Flash mode: This example can run on Hitex Flash (external parallel flash SST39X320X), SPIFI quad SPI flash (with or without debug)
				Select "Project->Download->Download active application" to burn into flash

	Step to run:
		- Step 1: Build example.
		- Step 2: Burn hex file into board (if run on Hitex Flash mode)
		- Step 3: Connect UART0 port on this board to COM port on your computer
		- Step 4: Configure hardware and serial display as above instruction 
		- Step 5: Run example, using Serial display to monitor the message and to send a charater to UART

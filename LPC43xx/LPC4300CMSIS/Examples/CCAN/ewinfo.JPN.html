Example description

  
C_CAN peripheral test example

Example description:
	Purpose:
		This example describes how to test C CAN peripheral of LPC43xx
	Process:
		This demo need a CAN analyser or, another CAN device on bus.
		Set PLL1 clock = 72MHz  in CGU_Init
		After init, the lpc43xx will transmit first message with ID 0x200 on bus.
		If success, it will wait for any trigger on wake up source 0 and then start transmit again.
		The device always listen message on the bus with ID 0x100. Whenever it receive,
		it will reply back the message to bus with ID 0x200.

Directory contents:
	\IAR:	includes IAR EWARM project and configuration files
	lpc43xx_libcfg.h: Library configuration file - include needed driver library for this example
	CCan_SimpleTxRx.c: Main program

How to run:
	Hardware configuration:
		This example was tested only on:
			Hitex LPC4350 Board

		Serial display configuration:(e.g: TeraTerm, Hyperterminal, Flash Magic...)
			– 115200bps
			– 8 data bit
			– No parity
			– 1 stop bit
			– No flow control
		Jumpers (to switch COM to port 0):
			SV1: 1-2, 3-4 close
			SV9: 5-6, 7-8 open
			SV10: 5-6, 9-10 open

	Running mode:
		RAM mode:   This example can run on RAM mode.
		ROM mode:   This example can run on EXFLASH mode.

	Step to run:
		- Step 1: Build example.
		- Step 2: Connect UART1 on this board to COM port on your computer
		- Step 3: Configure hardware and serial display as above instruction
		- Step 4: use a CAN analyser and set bitrate to 500kb/ss Start capture the bus
		- Step 5: Create new transmit Message by Go to Transmit->New Message
					Type in ID = 0x100, DLC = 8 and dont check anything
					Type whatever data you want to transmit, device will feedback it
		- Step 6: Run example and observe result on serial display

		(Pls see "LPC43xx Example Description" document - chapter "Examples > C_CAN > SimpleTxRx"
		for more details)

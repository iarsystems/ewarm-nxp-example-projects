@Example description:
	Purpose:
		This example describes how to use ADC conversions and transfer converted data by using DMA.
	Process:
		Because their is no potentiometer on EA board, so we have to connect ADC port to external potentiometer module
		The ADC conversion rate is 200KHz. A fully A fully accurate conversion requires 11 of these clocks.
		So ADC clock = 200KHz * (number of clocks for 1 measure = 11 clocks).
		Note that maximum ADC clock input is 4.5MHz.

		ADC will generate interrupt at the end of each conversion. It will make a request for DMA source to transfer
		the converted data from ADC data register "AD0DR" to "adc_value" variable.
		After display this data into serial display via UART0, DMA re-setup for next transfer.

		Turn potentiometer to change ADC signal input.

@Directory contents:
	\IAR:	includes IAR EWARM project and configuration files
	lpc43xx_libcfg.h: Library configuration file - include needed driver library for this example
	Adc_Dma.c: Main program file

@How to run:
	Hardware configuration:
		This example was tested on:
			Hitex LPC4350 Board

		ñ 115200bps
		ñ 8 data bit
		ñ No parity
		ñ 1 stop bit
		ñ No flow control

	Running mode:
		RAM mode:   This example can run on RAM mode.
		ROM mode:   This example can run on EXFLASH mode.

	Step to run:

		- Step 2: Connect UART1 on this board to COM port on your computer
		- Step 3: Configure hardware and serial display as above instruction
		- Step 4: Start Debug example, Run it, turn potetiometer and observe data on serial display

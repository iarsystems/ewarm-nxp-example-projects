@Example description:
	Purpose:
		This example demonstrates inter-processor communication between the Cortex-M4
		and Cortex-M0 cores in the LPC4350 CPU.
	Process:
		- M4: Configure Cortex M4 & system clock to 204 MHz
		- M4: Copy Cortex M0 image into ram
		- M4: Start Cortex M0 image
		- M4: Wait for messages
		- M0: Init UART, Start main loop and send / process messages
		- M4: Process messages
		- M0: Process messages

@Directory contents:
	\IAR:	includes IAR EWARM M0 and M4 project and configuration files
	\M0:	includes M0 code
	\M4:	includes M4 code

@How to run:
	Hardware configuration:
		This example was tested on:
			Hitex LPC4350 Board

	Step to run:
		- Step 1: Build example. First Build M0 project, then build M4 project. Clicking right
		   over M0/M4 allows selection of the Active Project in the workspace.
		- Step 2: Burn hex file into board (if run on ROM mode)
		- Step 3: Connect UART1 port on this board to COM port on your computer
              + Open any terminal program.
              + Chose 115200 baud, parity none, 8 data bits, 1 stop bit and open the COM port.
              + Run the example (Only M4 project. M0 project is linked to M4 code).
              + See communication between two cores on the terminal screen.

	Running mode:
		RAM mode:   This example can be run on RAM mode with debugger.
		ROM mode:   This example can be run on EXFLASH mode with debugger.

EWARM note: 
		M0 Application always executes in RAM (in EWARM use "Debug without Download", if you want to debug it).

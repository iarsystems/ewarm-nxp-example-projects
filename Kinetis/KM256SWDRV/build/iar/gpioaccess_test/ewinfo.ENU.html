Example description

  
DESCRIPTION
===========
   The Software Examples demonstrate correct use of the MKM34Z256VLx7 bare metal
  software drivers in an application. 
  This example shows use of the General-Purpose Input/Output (GPIO) port protection
  mechanism. The GPIOA is configured to allow RW privileged and user 
  secure/non-secure accesses. On the contrary GPIOB is configured for privileged RW
  accesses only (user secure/non-secure RW accesses are not allowed). Consequently,
  GPIO_Tgl(FGPIOB,PIN0) and GPIO_Tgl(GPIOB,PIN0) macros don't execute in a user mode
  to insufficient access level. The IOPORT accesses, demonstrated by
  GPIO_Tgl(FGPIOB,PIN0) macro, that are not allowed are treated as a RAZ/WI (Read As
  Zero/Write Ignored). On the contrary accesses via Peripheral Bridge, demonstrated by
  GPIO_Tgl(GPIOB,PIN0) macro, which are not allowed, terminate by the Hard Fault. 
 
CONFIGURATION
=============
  
  Release: The example is flashed into the embedded flash memory of the KM34Z256
  microcontroller.
  
  By default the project is configured to use an I-Jet debugger.
  
 For more details please read the 
 MKM34Z256VLx7 Bare Metal Software Drivers Reference Manual.

Example description

  
DESCRIPTION
===========
   The Software Examples demonstrate correct use of the MKM34Z256VLx7 bare metal
  software drivers in an application. 
  This software example demonstrates typical use of the Flash Memory Module (FTFA).
  The software initializes nvm_data[] flash array, placed by linker at address 0x1fc00
  by four unsigned long values. Then FTFA_EraseSector() function is called. It erases
  sector thus all 0s, within the 1KB sector pointed by address 0x1fc00, will
  transition to 1s. After flash sector has been erased, the function FTFA_WriteArray()
  is called. This function writes ram_data[] RAM array into flash at address 0x1fc00.
 
  Warning: A flash memory location must be in the erased state before being
           programmed. Cumulative programming of bits (back-to-back program operations
		   without an intervening erase) within a flash memory location is not
		   allowed. Re-programming of existing 0s to 0 is not allowed as this
		   overstresses the device.
		   
CONFIGURATION
=============
  
  Release: The example is flashed into the embedded flash memory of the KM34Z256
  microcontroller.
  
  By default the project is configured to use an I-Jet debugger.
  
 For more details please read the 
 MKM34Z256VLx7 Bare Metal Software Drivers Reference Manual.

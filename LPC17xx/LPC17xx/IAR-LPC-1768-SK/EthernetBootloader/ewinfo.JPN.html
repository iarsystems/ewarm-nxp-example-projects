Example description

  
Ethernet Bootloader

DESCRIPTION
===========
  This example shows how to use the IAR Embedded Workbench for ARM to develop code for IAR LPC-1768-SK board.
 The Ethernet bootloader enable an use program to be loaded to microcontroller's flash using Ethernet. The first
 8KB are used by the bootloader. The rest of the flash can be used by the user program.
 
 On start up the bootloader will check for valid user program at address 
 0x00002000. If there is no user code or B1 button is pressed (if CRP3 is 
 selected the bootloader will not check for pressed button) the Ethernet
 bootloader is starded. Else the User program is started.
  
 User Program Build:
  In order to be able to load user program with the Ethernet bootloader you should do:
  1. Link the user program at start address 0x2000. You can do this by simply
     using the EthernetBootlaoder\config\LPC1768_UserFlash.icf file.
     
  2. Build a hex image of the user program: Select Project\Options\Output Converter
     Check the "Generate additional output" box and select "Intel extended"
   

 To download a user program:
  1. Program the Ethernet Bootloader example on the target board (IAR-LPC1768-SK). 
  2. Reset the board with B1 button pressed.The Bootloader is now started and LED1 will be on.
  3. Connect a LAN cable to the board. The LED1 will blink.
  4. Start Flash Magic program on PC connected to the same LAN.
  5. Configure the Flash Magic program. Basically you should set the device, the 
     MAC address and IP address (Please read NXP application note AN10744 and Flash Magic Manual for 
     more details).
  6. Browse to select the user example hex file
  7. Press "Start". The program is now downloaded to the 
     microcontroller's flash.
  
  8. Reset the board. The user program should start.
  
  To load new user program repeat from step 2.

Note: If CRP3 is selected, no future factory testing can be performed on the device
 
COMPATIBILITY
=============

   The example project is compatible with IAR LPC-1768-SK evaluation board.
  By default, the project is configured to use the J-Link JTAG interface.

CONFIGURATION
=============

   After power-up the controller get clock from internal RC oscillator that
  is unstable and may fail with J-Link auto detect. In this canse adaptive
  clocking should be used. The adaptive clock can be select from menu:
  Project->Options..., section Debugger->J-Link/J-Trace  JTAG Speed - Adaptive.

   Make sure that the following jumpers are correctly configured on the
  IAR LPC-1768-SK evaluation board:

  PWR_SEL     - depending of power source
  RST_E       - unfilled
  ISP_E       - unfilled

GETTING STARTED
===============

  1) Select File->Open->Workspace...
     Open the following workspace:

     My Documents\IAR Embedded Workbench\arm\examples\NXP\
     LPC17xx\IAR-LPC-1768\EthernetBootloader\Bootloader.eww

  2) Power on the board and connect the J-Linck.

  3) Press Ctrl+1 or use "Download active application" to build the project and 
     download it to the board.

  4) Reset the board and meanwhile hold the B1 pressed. The Bootloader is now 
     started and LED1 will be on.
  
  5) Connect a LAN cable to the board. The LED1 will blink.

  6) Start Flash Magic program on PC connected to the same LAN.

  7) Configure the Flash Magic program. Basically you should set the device, the 
     MAC address and IP address (Please read NXP application note AN10744 and Flash Magic Manual for 
     more details)

 11) Browse to 
        My Documents\IAR Embedded Workbench\arm\examples
        \NXP\LPC17xx\IAR-LPC-1768\EthernetBootloader\hex
     
     and select one of the prebuild hex files.
     
 12) Press "Start". The program is now downloaded to the 
     microcontroller's flash.

 13) Reset the board. The user example starts.
